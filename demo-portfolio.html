Multi-Auction Portfolio Optimization

5개의 광고 지면에 총 예산을 어떻게 나눠야 전체 수익이 최대가 될까? 각 지면의 Diminishing Returns 곡선을 기반으로 한 포트폴리오 최적화를 슬라이더로 체험합니다.

시나리오 — 일일 총 예산이 고정되어 있고, 5개의 서로 다른 광고 지면(Slot A~E)이 있습니다. 각 지면은 고유의 True Value와 Saturation 상수 K를 갖습니다. 수익 함수는 Profiti(Bi) = Vi × (1 − e−Bi/Ki) × Ki − 0.3 × Bi 형태로, "처음엔 큰 이익 → 점점 diminishing returns → 결국 포화" 구조입니다.

핵심 질문: 고가치 지면 하나에 몰아주는 게 좋을까, 여러 지면에 분산하는 게 좋을까? 등한계효용 원리(Marginal Utility Equalization)로 답을 직접 발견해 봅니다.

✨ 이 데모에서 알게 된 3가지

  1. 고가치 지면에 몰빵 ≠ 최적 — Diminishing Returns 때문에 같은 지면에 계속 붓다 보면 한계 수익이 0에 수렴. 다른 지면의 초반 큰 기울기를 놓치게 된다.
  2. 등한계효용 원리 — 각 지면의 한계 수익(Profit의 기울기)이 모든 지면에서 같아지도록 분배할 때 전체 Profit이 최대. "Auto Optimize" 버튼이 이 조건을 수치적으로 찾아낸다.
  3. 이것이 Budget Pacer의 본질 — 실제 광고 시스템의 Pacing 알고리즘(PID·Lagrangian Dual)도 결국은 이 포트폴리오 문제를 시간과 지면 두 축으로 동시에 푸는 것.
