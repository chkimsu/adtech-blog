사용자가 페이지를 연 순간부터 광고가 뜨기까지 — 요청·입찰가·소재·데이터가 어디로 흐르는지, 19개 모듈을 왼쪽(사용자)에서 오른쪽(광고주)으로 읽는 순서대로 따라가 봅니다.

아래에서 ① 흐름을 고르고 ▶ 재생하면, 지도 위 라벨 패킷이 움직이며 각 단계의 설명과 예시가 옆 패널에 차곡차곡 쌓입니다. 노드를 클릭하면 정의·데모·포스트도 열려요.

지도 따라가기

왼쪽(사용자)에서 오른쪽(광고주)으로 읽습니다. 색은 역할, 화살표는 이동 방향. 재생을 누르면 라벨 패킷이 움직이고 옆 패널에 단계 설명·예시가 쌓여요.

  • 색 = 역할. 보라(사는 쪽)·금색(파는 쪽)·회청(거래소)·적갈(ML 모델)·녹색(측정)·황토(사용자) — 아래 범례와 같습니다.

  • ▶ 버튼 = 시나리오 재생. 알약 모양 패킷이 "지금 무엇이 이동 중인지"를 라벨로 보여주고, 옆 패널에 단계별 설명이 쌓입니다. 일시정지·이전·다음 버튼으로 천천히 봐도 됩니다.

    6 카테고리로 묶은 광고 생태계

    각 카테고리는 광고 거래의 한 축을 담당합니다. 위 그래프의 색과 동일.

    Buy Side · 5개 모듈

    광고주가 광고를 사는 쪽. Advertiser → Brand/Agency → DSP가 캠페인을 실행하고, DMP/CDP가 타겟팅 데이터를, DCO가 크리에이티브를 자동 생성한다.

    Exchange · 2개 모듈

    거래소. Ad Exchange가 광고 요청을 받아 여러 DSP에 동시 Bid Request를 뿌리고, Auction Engine이 1st/2nd Price 규칙으로 낙찰가를 결정한다.

    Sell Side · 3개 모듈

    매체가 지면을 파는 쪽. Publisher가 광고 슬롯을 제공하고 SSP가 그걸 Exchange에 판다. Header Bidding이 Waterfall 대비 +10~30% 더 비싸게 파는 기술.

    ML / Models · 4개 모듈

    예측·랭킹의 두뇌. Feature Store가 피처를 공급하고 Model Serving이 수천 후보를 좁힌다. pCTR/pCVR이 확률을 예측, Calibration이 절댓값을 보정한다.

    Measurement · 2개 모듈

    측정·피드백. Log Pipeline이 Impression·Click·Conversion을 모두 수집하고, MMP/Attribution이 "어느 광고가 전환을 만들었나"를 판정한다.

    User / Privacy · 2개 모듈

    최종 소비자와 데이터 보호. User Journey는 노출→클릭→전환의 시간 흐름, CMP는 GDPR 동의, Walled Garden은 닫힌 생태계 (구글·메타).
